# 开发记录：第二纪元入口提示的探索友好化收敛

**日期:** 2026-06-09
**阶段:** 第二纪元入口体验微调
**状态:** 已落地

---

## 1. 背景

在上一轮奇点入口引导增强后，系统已经能更清楚地回答：

1. 为什么现在还不能点技术奇点
2. 第一次奇点还缺 CP 还是缺筹备度
3. 第二纪元刚开启时有哪些关键系统

但其中一部分文案开始越过“帮助理解规则”的边界，变成了偏明确的“推荐路线”。

这会带来两个风险：

1. 首次进入第二纪元时，玩家容易直接照抄，而不是自己读图和试错
2. 一旦默认路线写得太具体，探索感和分支选择的乐趣会被稀释

---

## 2. 这次调整的边界

本次不是回退引导，而是把引导改成更中性的“规则说明”。

保留：

1. 条件状态
2. 系统规则
3. 可逆性与代价信息

收敛：

1. 明确点名的首选路线
2. 带明显倾向的“你现在应该这样打”式建议
3. 容易让玩家把系统当成标准答案的路线提示

---

## 3. 已落地改动

### 3.1 技术奇点确认弹窗

移除了“首次奇点后的推荐开局”。

改为说明：

1. SP 会购买编程范式
2. 这些购买会跨越后续重构与编译发布保留
3. 第二纪元一开始就存在学派限制与技术债务，但技术债务重规划代价很高

### 3.2 编程范式页顶部说明

移除了“首次 3 SP 推荐路线”。

改为显示第二纪元基础规则：

1. 已购范式会跨循环保留
2. 三大学派最多同时发展两条
3. 技术债务可以返还被冻结的 SP，但成本昂贵

### 3.3 架构过载反馈

将原本偏动作指导的“当前建议”改成“当前状态”：

1. 解释过载已经生效
2. 解释继续堆 AI Core 的边际收益会下降
3. 在接近奇点时，把过载描述为最后冲刺阶段的效率折损

同时，首页点击过载提示后的 toast 也改为：

1. 告知当前效率
2. 引导查看统计页
3. 不再直接替玩家下操作结论

### 3.4 技术债务面板文案

保留“还没还清 / 已可确认”的状态区分，但把提示改成更客观的描述：

1. 未满足时，强调还需要继续累积 CP
2. 满足时，强调会返还被冻结的 SP 并重新开放选择

---

## 4. 当前结论

这一轮调整后的入口信息，更接近我们想要的平衡点：

1. 玩家不会因为规则不透明而迷路
2. 也不会因为系统写出标准路线而失去探索空间

它本质上是在把“教程”从攻略，收回到规则说明。

---

## 5. 后续建议

如果后续继续补第二纪元或第三纪元的引导，建议沿用这条边界：

1. 首次出现的新系统，可以解释规则、限制、代价和可逆性
2. 非必要时，不直接写最优路线、首选节点或默认 Build
3. 真正偏攻略的信息，更适合放在外部文档或玩家主动打开的帮助页，而不是默认贴在核心流程 UI 上
