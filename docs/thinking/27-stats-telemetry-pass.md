# 统计页遥测补强记录

## 本轮目标

让“统计”页从当前状态面板，升级为能反映存档成长轨迹的长期统计页。

## 本轮新增

- 总游玩时长：只统计前台真实运行时间，不把离线时间混进“游玩时长”。
- 累计离线收益时长：单独统计玩家实际结算过的离线收益时间。
- 手动编写次数：反映玩家主动操作强度。
- 累计产出算力：统计手动点击、在线生产、确认领取的离线收益所获得的总 CP。
- 累计重构点：统计历史总获得 RP，而不是当前持有 RP。
- 总奇点次数：复用现有 `singularityCount`，补到统计页中。
- 已完成挑战：基于挑战完成状态汇总展示。

## 设计取舍

- “总游玩时长”没有复用离线收益时长，因为离线推进代表存档在增长，不代表玩家正在游玩。
- “累计离线收益时长”只在玩家确认领取离线收益时增加，避免弹窗预览阶段就提前记账。
- “累计产出算力”在离线收益确认时才记账，避免只弹出离线提示但未领取时提前计入。
- “总重构点”改为累计口径，是为了避免和“当前版本”“当前效率”混用时产生理解偏差。

## 这轮先不加的统计

- 最高 CPS：适合后续做“峰值表现”模块时再加。
- 单局最长运行时长：需要额外定义“单局”边界，当前先不引入。
- 各生成器总购买量：价值存在，但当前统计页信息密度已经足够高。
- 广告增益使用次数：广告系统尚未落地，暂不提前埋点。
